# `/stock-guide` ŌĆö sub-PRD

Owner: `worker_dash-stock-guide`. Reports to `worker_subgerente-app`.

> **Visual identity note:** Stock Guide is **NOT** the Market Watch / trading-terminal module. It uses the **standard dashboard identity** (brand orange `#ff5000`, Arial, liquid-glass cards) exactly like `/subsidy-tracker` and the other Fase-3 dashboards ŌĆö never the scoped `.stocks-dark` / `.stocks-light` theme owned by `/stocks`. The only thing it borrows from Market Watch is the Yahoo quote proxy (`/api/stocks/quote`) and the `useStockQuote` hook.

## Overview

Equities-research module for the Brazilian Oil & Gas + Fuel Distribution coverage universe. Two analyses, one shared brain:

1. **Comps table** ŌĆö one row per covered company with the headline research numbers: target price, recommendation (OP/MP/UP), and six forward column pairs (EV/EBITDA, P/E, FCFE Yield, Div Yield, EBITDA, Volumes), each split into two forward years (`config.y1_label` / `config.y2_label`, e.g. 2026E / 2027E). **Market cap**, **upside**, and the **four price-sensitive multiples (EV/EBITDA, P/E, FCFE Yield, Div Yield)** are computed LIVE (never stored) ŌĆö see below. Only EBITDA and Volumes are direct data.
2. **Per-company sensitivity** ŌĆö a freeform 2D matrix (admin-defined axes/labels/cells) for the selected company, opened below the table (desktop) or in a BottomSheet (mobile).

Targets, shares, the **fundamentals** (net debt, EBITDA, net income, FCFE, dividends), volumes, sensitivities and config are **admin-only inputs** maintained in the Admin Panel ŌåÆ Stock Guide section. The four price-sensitive multiples are **NOT stored** ŌĆö they are always derived live from the price + fundamentals (no admin override). Any company can be **hidden** (regulatory restriction); hidden companies are removed from the table and listed only by name in a "Currently restricted" footnote ŌĆö their financials never reach a non-admin browser.

Audience: **Client + Admin**, hidden from anonymous public (`is_visible_for_public=false`). One-click reconfigurable in Admin Panel ŌåÆ Permissions.

Slug / URL: `stock-guide` ŌåÆ `/stock-guide`. NavBar placement: **standalone top-level NavBar entry** ("Stock Guide", alongside Market Watch / News Hunter), not under any dropdown.

## Live derivations (frontend, in the hook)

Computed per visible row from the batched Yahoo quote + the admin-input fundamentals ŌĆö **not** stored server-side. **All monetary inputs are BRL million**, so EV/EBITDA and P/E are dimensionless and the yields are ├Ś100 for percent points.

| Field | Formula | Null-safety |
|---|---|---|
| `livePrice` | `quote.regularMarketPrice` matched on `yahoo_symbol` (fallback `ticker`) | `null` if the quote is missing |
| `marketCapBrlMn` | `shares_outstanding ├Ś livePrice / 1e6` (BRL mn) | `null` if either input is `null` |
| `upsidePct` | `target_price / livePrice ŌłÆ 1` | `null` unless `livePrice > 0` and `target_price` present |
| `evBrlMnY1` / `evBrlMnY2` | `marketCapBrlMn + net_debt_yN` (BRL mn). Net debt is **forward, per year**; either may be negative (net cash ŌåÆ lowers EV). Market cap stays a single current value. | `null` if either input is `null` |
| `evEbitdaY1` / `evEbitdaY2` | `evBrlMnYN / ebitda_yN` | `null` unless `ebitda_yN > 0` (EBITDA Ōēż 0 ŌåÆ not meaningful ŌåÆ `ŌĆö`) |
| `peY1` / `peY2` | `marketCapBrlMn / (net_income_adj_yN ?? net_income_yN)` ŌĆö uses the ADJUSTED net income when present (e.g. Vibra, which strips non-recurring tax credits), else falls back to the reported net income | `null` unless the chosen earnings `> 0` (non-positive earnings ŌåÆ P/E not meaningful ŌåÆ `ŌĆö`) |
| `fcfeYieldY1` / `fcfeYieldY2` | `(fcfe_yN / marketCapBrlMn) ├Ś 100` percent | `null` unless `marketCapBrlMn > 0`; FCFE may be negative ŌåÆ negative yield shown |
| `divYieldY1` / `divYieldY2` | `(dividends_yN / marketCapBrlMn) ├Ś 100` percent | `null` unless `marketCapBrlMn > 0` |

Everything renders `ŌĆö` (never `NaN`) when null. Every divide-by-zero / non-positive denominator is guarded. Market cap and EV land in BRL because `shares_outstanding` is the absolute share count and the live price is in BRL; net debt and the per-year fundamentals are all BRL mn. **EV convention:** `EV(year) = Market cap + Net Debt(year)` ŌĆö net debt is now **forward per year** (`net_debt_y1` / `net_debt_y2`), so EV is computed per forward year. Market cap remains a single current value. **No override:** the four multiples are *always* computed ŌĆö there is no admin-stored fallback.

The 4 live multiples render `ŌĆö` while `quotesLoading` (they depend on the live price); EBITDA and Volumes are direct data and never gate on the quote.

### Live-quote wiring & cadence

- The hook collects the `yahoo_symbol` (fallback `ticker`) of **visible** rows into one de-duplicated list and passes it to `useStockQuote(symbols)` ŌåÆ a single batched `GET /api/stocks/quote?tickers=ŌĆ”` request (the proxy auto-appends `.SA` for B3 tickers and returns an array).
- Cadence: **one-shot fetch on load + a manual "Refresh quotes" button** (desktop only). **No polling ticker** ŌĆö comps are snapshots and the Yahoo proxy is per-IP rate-limited.
- Hidden companies have a `null` `yahoo_symbol` AND are excluded from `visibleRows`, so a restricted ticker is structurally absent from the quote request ŌĆö the browser cannot fetch a restricted name's price even if it tried.

## Hide-aware RLS posture

The three tables have **RLS enabled with NO SELECT policy and NO write policy** (deliberately stricter than `field_stakes`). With RLS on + zero policies, direct PostgREST returns **0 rows** for non-superusers ŌĆö `supabase.from('stock_guide_companies').select('*')` yields `[]` for anon/client. All access flows through SECURITY DEFINER RPCs, which is what keeps hidden financials server-side.

`get_stock_guide_comps()` is the hide-aware core: it returns one row per company in `display_order`, but for hidden rows seen by a non-admin it forces **every** field except `ticker` / `company_name` / `is_visible` / `display_order` to NULL (including `sector`, `volume_unit`, `yahoo_symbol`, `shares_outstanding` and all comps). Admins (`is_admin()`) receive every field through the same call. `get_stock_guide_sensitivity(ticker)` returns `{}` for a hidden company unless the caller is an admin.

### Dynamic drivers (live market-data catalog)

Source of truth: `src/hooks/useMarketDrivers.ts` (frontend-only ŌĆö no DB compute). DB `source` column was added by `worker_supabase` (commit `8c3c084f`, migration `20260607000000`).

A driver in `stock_guide_drivers` is now either:

- **Static** ŌĆö `source` is null/`''`; the admin types `current_value`; the effective "today" value is `current_value`.
- **Dynamic** ŌĆö `source` is a key in the **market-driver catalog** below; `current_value` is ignored (may be null) and the effective value is **computed LIVE in the browser** from the Yahoo proxy. The unit is auto-filled from the catalog.

**Catalog** (`MARKET_DRIVER_CATALOG`, 4 metrics):

| `source` key | Label | Unit | Computation |
|---|---|---|---|
| `avg_brent_2026` | Avg Brent 2026 | USD/bbl | month-average of 12 (realized + forward, see below) |
| `avg_brent_2027` | Avg Brent 2027 | USD/bbl | month-average of 12 (all forward, fallback spot) |
| `avg_fx_2026` | Avg FX (USD/BRL) 2026 | BRL/USD | month-average of 12 (realized + **spot-flat** approx) |
| `avg_fx_2027` | Avg FX (USD/BRL) 2027 | BRL/USD | spot, held flat (= spot) |

**Inputs** (3 GETs to the existing proxy, **once on mount**, no polling ŌĆö these are slow-moving macro assumptions and the proxy is per-IP rate-limited):

- `GET /api/stocks/futures-curve` ŌåÆ `contracts[{ month /*0-indexed*/, year, price }]` ŌĆö monthly Brent forward curve (~24 contracts, starts ~M+2, server-cached 24h).
- `GET /api/stocks/history?ticker=BZ=F&range=1y&interval=1d` ŌåÆ Brent realized daily history `[{ date /*unix sec*/, close }]`.
- `GET /api/stocks/history?ticker=USDBRL=X&range=1y&interval=1d` ŌåÆ USD/BRL realized daily history.
- `GET /api/stocks/quote?tickers=BZ=F,USDBRL=X` ŌåÆ spot fallbacks (`regularMarketPrice`; fallback to last realized close).

**Computation** (uses the REAL current date `new Date()` in the browser; `now=new Date()`, `curYear=now.getFullYear()`, `curMonth=now.getMonth()+1` Ōłł 1ŌĆō12). Helpers: `realizedMonthlyAvg(history, year, month)` = mean of `close` over candles whose **UTC** date is in {year, month} (null if none); `forwardPrice(curve, year, month)` = price of the contract with `year === c.year && month === c.month+1` (null if absent / non-positive).

- **`avg_brent_<Y>`** ŌĆö mean of the 12 monthly values, each by month `m=1..12`:
  - past (`Y<curYear` OR (`Y===curYear` AND `m<curMonth`)) ŌåÆ `realizedMonthlyAvg(brentHist, Y, m)`;
  - current (`Y===curYear` AND `m===curMonth`) ŌåÆ `realizedMonthlyAvg(brentHist, Y, m) ?? brentSpot` (month-to-date, falls back to spot);
  - future ŌåÆ `forwardPrice(curve, Y, m) ?? brentSpot` (the near 1ŌĆō2 months may not be in the curve since it starts ~M+2 ŌåÆ fall back to spot).
  - Result = mean of the non-null monthly values. For Y=2027 every month is future.
- **`avg_fx_<Y>`** (spot-flat approximation ŌĆö there is no FX forward in the proxy): past ŌåÆ `realizedMonthlyAvg(fxHist, Y, m)`; current + future ŌåÆ `fxSpot` held flat. For Y=2027 every month ŌåÆ `fxSpot`, so the mean = `fxSpot`.

**Null-safe throughout:** if the inputs are missing, the metric value is `null` and the UI renders "ŌĆö".

**Wiring.** `useMarketDrivers()` returns `{ values: Record<key, number|null>, loading, error, catalog }` plus two exported helpers used by both consumers:

- `resolveDriverValue(driver, marketValues)` ŌåÆ the driver's effective today value (`marketValues[source]` when `source` is a catalog key, else `driver.current_value`).
- `isDynamicSource(source)` ŌåÆ true when `source` references a known catalog key.

The hook is reused by **both** the dashboard brain (`useStockGuideData`) and the admin-panel Drivers editor, so the same numbers drive the sensitivity-table highlight AND the admin "Computed: ŌĆ”" preview. In `useStockGuideData`, `resolveDriverAxis(axis)` now also returns a resolved `currentValue` (via `resolveDriverValue`); both Views highlight / interpolate / caption on this `currentValue` instead of the raw `driver.current_value`, so a dynamic driver tracks the live market value. `marketValues` + `marketDriversLoading` are exposed on the hook for raw-map consumers.

**Admin Drivers editor** (`admin-panel`, desktop-only): each driver row has a **Source** select ŌĆö **Static** (default, `source=''`) + the 4 catalog metrics by label. Picking a catalog metric sets `source`, auto-fills `unit` from the catalog (unit input disabled), disables the `current_value` input, and shows the live "Computed: {value} {unit}" (FX metrics carry a "(spot-based approximation ŌĆö no FX forward)" note). A "dynamic ┬Ę {value}" badge renders under the name. On save, a dynamic driver sends `source` + catalog `unit` + `name` + `display_order` with `current_value: null`; a static driver is unchanged (`source: ''`). The `rpcAdminUpsertStockGuideDriver` wrapper is generic ŌĆö it forwards `source` untouched.

### Hook (`useStockGuideData.ts`)

- Fetches drivers + sensitivity tables in the same batched `Promise.all` as comps + config. Exposes `drivers: StockGuideDriver[]`, `sensitivityTables: SensitivityTable[]`, and derived `selectedTables` = tables where `selectedTicker Ōłł table.companies` (sorted by `display_order`). `selectedTicker`/`selectTicker` kept (default = first visible company); the per-table fetch is gone (tables arrive in the initial batch).
- `computeSensitivityCell(table, rowIdx, colIdx) ŌåÆ { value, unit }` ŌĆö the live-derived cell helper. Resolves the cell's company (row company axis ŌåÆ `companies[rowIdx]`; else col company axis ŌåÆ `companies[colIdx]`; else single-company `table.companies[0]`), looks up that company's live `livePrice` + `marketCapBrlMn` from a `liveByTicker` index (built from `visibleRows`, NOT the sector-filtered `computedRows`, so a cell always resolves even when the sector filter hid that company), then applies `value_mode`: `absolute`ŌåÆprimary; `yield`ŌåÆ`primary/marketCap*100` (guard marketCap>0); `pe`ŌåÆ`marketCap/primary` (guard primary>0); `ev_ebitda`ŌåÆ`(marketCap+secondary)/primary` (guard primary>0, secondaryŌēĀnull, marketCapŌēĀnull); `upside`ŌåÆ`(primary/livePriceŌłÆ1)*100` (guard livePrice>0). Null-safe ŌåÆ `null` ŌåÆ render "ŌĆö". The returned `unit` is the table's own unit for `absolute`, `'%'` for yield/upside, `'├Ś'` for pe/ev_ebitda. `formatSensitivityCell(value, unit)` (exported) formats it.
- `resolveDriverAxis(axis) ŌåÆ { driver, scenarios }` ŌĆö looks up `driver_id` in `drivers`.

## Scenario-grid sensitivity ŌĆö multi-axis Brent mesh (1..3 axes, multilinear)

> **Multi-axis redesign (2026-06-18) of the 1-D Brent grid.** The 1-D mesh had a single ambiguous axis (`x_value` = "Brent of which year?"). It is now a REGULAR mesh over **1..3 driver axes** (e.g. Avg Brent 2026 / 2027 / 2028+): the analyst runs their model over the FULL Cartesian product of per-axis levels and uploads, PER COMPANY, the target price at every mesh node; the frontend interpolates that mesh **MULTILINEARLY** (a 2^d corner blend ŌĆö linear / bilinear / trilinear) as the analyst drags **one slider per axis**. Clean break ŌĆö no dual-path 1-D code survives in the frontend; the DB migration `ALTER`ed the table and converted any legacy shape.

Source of truth: `supabase/migrations/20260612000000_stock_guide_scenario_grid.sql` (original table + RPC) + `supabase/migrations/20260618200000_stock_guide_scenario_grid_multi_axis.sql` (the multi-axis generalization: `y_value`/`z_value` columns, 5-col PK, 5-col RPC, legacy cleanup; owner `worker_supabase`) + the frontend below. A sensitivity table is a **SCENARIO GRID** when its `definition` carries a **`grid`** block; the dashboard then renders a multi-slider panel that interpolates the **target price + upside** live as the analyst drags, instead of the static matrix. Static tables (no `grid`) are unchanged.

### Data model ŌĆö the mesh table + the grid block

- **`stock_guide_scenario_grid(sensitivity_id bigint, ticker text, x_value numeric, y_value numeric, z_value numeric, primary_value numeric)`** ŌĆö PK `(sensitivity_id, ticker, x_value, y_value, z_value)`, FK ŌåÆ `stock_guide_sensitivities(id)` `ON DELETE CASCADE`. `x/y/z_value` are the driver levels (one per `definition.grid.axes` entry, in order; an UNUSED axis is always `0` ŌĆö `NOT NULL DEFAULT 0`); `primary_value` = target price (BRL/share) at that mesh node. **The SENSITIVE per-company points live here, NOT in the jsonb.** RLS enabled, **no policies** (same posture as every `stock_guide_*` table) ŌĆö reads via the hide-aware RPC, writes via the service-role Excel-upload pipeline (replace-total snapshot per `sensitivity_id`, bypasses RLS). The 5-col PK btree covers the canonical read (filter by `sensitivity_id`, per-ticker series ordered by the coordinate axes).
- **`definition.grid`** (metadata only, stored verbatim by the existing `admin_upsert_stock_guide_sensitivity_table`):

```jsonc

    { "driver_key": "avg_brent_2026", "label": "Brent (avg 2026)", "unit": "USD/bbl" },
```

### RPC (`get_stock_guide_scenario_grid(p_sensitivity_id bigint)`)

`RETURNS TABLE(ticker text, x_value numeric, y_value numeric, z_value numeric, primary_value numeric)`, ordered by `ticker` then the coordinate axes, **hide-aware** (a non-admin only receives points for tickers flagged `is_visible`; admins get everything ŌĆö the criterion mirrors `get_stock_guide_sensitivity_tables`). `SECURITY DEFINER SET search_path = public, pg_temp`, GRANT `anon, authenticated`. Wrapper `rpcGetStockGuideScenarioGrid(supabase, sensitivityId)` in `src/lib/rpc.ts` coerces all four numerics ŌåÆ `number` and drops any row with a non-finite coordinate/value so the interpolator never sees `NaN`. `mapGridBlock` validates `axes` = an array of 1..3 `{driver_key, label, unit}` (non-empty `driver_key`; `label`/`unit` coerced to string; duplicate `driver_key` keeps the first; >3 dropped; **0 valid ŌåÆ `null` ŌåÆ the table falls back to a static matrix**).

### Interpolation (browser, pure ŌĆö `src/lib/stockGuideSensitivity.ts`)

Two pure helpers (unit-tested in `src/lib/__tests__/stockGuideSensitivity.test.ts`):

- **`buildGridMesh(points: MeshPoint[], dim): GridMesh | null`** ŌĆö collects the DISTINCT ascending levels per axis (exact equality ŌĆö both the stored coords and the live slider value flow from the same numeric column; the upload script rounds to 6 decimals), maps each point's coords ŌåÆ a tuple of per-axis indices, last-write-wins on a duplicate tuple, returns `null` for an empty cloud. `dim` is the table's axis count, so a degenerate axis (a single level) is still a real axis. `GridMesh = { dim, levels: number[][], values: Map<indexTupleKey, number> }` (key = `"i"` | `"i,j"` | `"i,j,k"`).
- **`interpolateMesh(mesh, at: number[]): number | null`** ŌĆö per axis: clamps a non-finite query to the axis min, brackets via binary search; a single level / `Ōēż first` ŌåÆ collapse at the lower edge (`frac 0`), `Ōēź last` ŌåÆ collapse at the upper edge, else `frac = (vŌłÆlo)/(hiŌłÆlo)` (span 0 ŌåÆ 0). It **COLLAPSES axes with `lo===hi` or `frac===0`** (it does NOT enumerate that axis's upper corner), so a border clamp on a complete mesh never demands a corner outside the active cell. It enumerates the `2^k` corners of the active cell (k = non-collapsed axes, Ōēż 3), `weight = ╬Ā (frac | 1ŌłÆfrac)`, and returns `╬Ż weight ┬Ę value`. **A required corner missing from the active cell ŌåÆ `null` (never `NaN`).**

**Upside** = `targetPrice / livePrice ŌłÆ 1` (guard `livePrice > 0`), reusing the dashboard's `liveByTicker` price index.

### Hook (`useStockGuideData.ts`)

The shared brain owns the grid state so **both Views drag the same sliders**:

- `isGridTable(table)` ŌĆö true when `definition.grid != null`.
- **Lazy mesh fetch + cache:** when a selected table is a grid, the hook fetches `rpcGetStockGuideScenarioGrid(table.id)` once (cached by id, guarded by a fetched-set ref) and stores the raw `ScenarioGridPoint[]`. `gridLoading` is true while a selected grid's mesh is in flight.
- **`gridIndexById` is built ONCE per fetch in a `useMemo`** (`Record<tableId, Map<ticker, GridMesh>>`, dimensioned by `axes.length`) ŌĆö NEVER per drag (the previous 1-D version rebuilt the series every render; that is fixed).
- Per-table axis overrides (`gridAxisValuesById: Record<id, (number|null)[]>`): `null` = follow the live value. **LAZY** ŌĆö never eager-initialised (market values arrive async), so a fresh axis stays `null` until dragged. `setGridAxisValue(tableId, axisIdx, v)`, `resetGridAxis(tableId, axisIdx)`, `resetGridAll(tableId)`.
- `getGridModel(table) ŌåÆ { axes: GridAxisModel[], rows, outputLabel, anyOverridden }` (or null for a non-grid table / while the mesh is empty). Each `GridAxisModel` = `{ key, label, unit, value, liveValue, min, max, step, disabled, overridden }` ŌĆö domain per axis = min/max of the UNION of that axis's levels across tickers; `step` = `gridSliderStep`; `liveValue` = `marketValues[driver_key]`; `value` = override ?? clamp(live) ?? midpoint; `disabled` = the union has a single level (no `min===max` padding hack); `label`/`unit` fall back to the catalog. `rows` = one MULTILINEARLY interpolated `{ ticker, companyName, targetPrice, upside, livePrice }` per visible ticker with a mesh, sorted by display order.
